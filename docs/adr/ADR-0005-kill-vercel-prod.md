# ADR-0005-kill-vercel-prod

Date: 2025-09-28

## Status
KILLED

## Decision
Использование Vercel для продакшн-деплоя фронтенда исключено из Scarlet Sails.

## Context
В проекте KIT_RnD_Layer4 Vercel использовался для деплоя фронтенд-приложений. Однако, упоминание о "feat/off-vercel-migration" в истории репозитория указывает на попытки миграции с платформы.

## Problem
- Потенциальные ограничения платформы, которые привели к попыткам миграции.
- Недостаточная гибкость или контроль над инфраструктурой.
- Возможно, проблемы с ценовой политикой или масштабированием.
- Отсутствие четкой долгосрочной стратегии выбора платформы деплоя.

## Consequences
- Для Scarlet Sails будет выбран более подходящий и долгосрочный вариант деплоя фронтенда, возможно, интегрированный с основным бэкендом на Render или другой платформе, обеспечивающей больший контроль.
- Выбор платформы будет обоснован архитектурным ревью и долгосрочной стратегией.

## Alternatives Considered
- Продолжить использование Vercel с учетом его ограничений - отклонено из-за предыдущих проблем.
- Использовать другую специализированную платформу для фронтенда - возможно, но требует тщательного анализа.
- Интегрировать фронтенд в бэкенд-деплоймент (например, на Render) - принято как предпочтительный вариант для MVP.

## Lessons Learned
- Выбор платформы деплоя должен быть обоснован долгосрочной стратегией, а не только удобством на начальном этапе.
- Необходимо учитывать потенциальные ограничения и стоимость масштабирования.
- Предпочтительна унификация деплоя фронтенда и бэкенда, если это не создает излишней сложности.
